=== OVERVIEW ===

This document describes our implementation of copy-on-write (CoW) fork
functionality in xv6. Instead of immediately copying all memory pages
during fork(), our implementation shares pages between parent and child
processes until one of them writes to a shared page.

=== PART A DEPENDENCY ===

Our CoW implementation depends on the enhanced scheduler from Part A in
several ways:

Process Management:
	- The scheduler handles process creation and cleanup properly
	- When CoW page faults occur, processes may need to yield CPU while
	  waiting for memory
	- The scheduler coordinates with memory management during process exit

Memory Allocation:
	- When kalloc() fails during CoW fault handling, the process may sleep
	- The scheduler ensures fair access to memory resources
	- Multiple processes writing to shared pages are handled correctly

Integration Points:
	- Process state transitions during fork() and exit()
	- Memory pressure handling when system is low on pages
	- Coordination between process scheduling and memory allocation

=== DESIGN APPROACH ===

Reference Counting:
	We added a reference count array that tracks how many processes share
	each physical page. When fork() is called, we increment the reference
	count instead of copying pages.

Page Sharing:
	Both parent and child processes map to the same physical pages, but we
	remove write permissions (PTE_W flag). When either process tries to
	write, a page fault occurs.

Page Fault Handling:
	The page fault handler (cowfault) checks if only one process owns the
	page. If so, it just restores write permission. If multiple processes
	share it, it allocates a new page, copies the contents, and updates
	the mapping.

=== IMPLEMENTATION DETAILS ===

Key Files Modified:
	- kernel/kalloc.c: Added reference counting system
	- kernel/vm.c: Modified uvmcopy() for sharing, added cowfault()
	- kernel/trap.c: Added page fault handling for CoW
	- kernel/proc.c: Updated process cleanup for proper reference
	  management

Data Structures:
	int page_refcount[NPHYS_PAGES];  // One count per physical page
	struct spinlock refcount_lock;   // Protects the reference counts

Core Functions:
	incref(pa) - Increment reference count for physical address
	decref(pa) - Decrement reference count  
	getref(pa) - Get current reference count
	cowfault(pagetable, va) - Handle copy-on-write page faults

Fork Process:
	1. Create new process structure as normal
	2. For each page in parent's address space:
	   - Increment reference count
	   - Map same physical page in child
	   - Remove write permission from both parent and child
	3. Return to user space

Write Fault Handling:
	1. Check if virtual address is valid
	2. Get reference count for the physical page
	3. If refcount == 1: just restore write permission
	4. If refcount > 1: allocate new page, copy data, update mapping
	5. Flush TLB to make changes visible

Process Exit:
	1. Unmap all pages in process address space
	2. For each page, decrement reference count
	3. Free physical page only if reference count reaches zero

=== MAJOR CHALLENGES ===

Reference Count Management:
	Initially had bugs where reference counts went negative. Fixed by
	centralizing all decrements in kfree() function.

Process Cleanup:
	Had "freewalk: leaf" panics during exit. Solved by using
	proc_freepagetable() which properly unmaps special kernel pages.

Page Fault Integration:
	Took time to understand xv6's trap handling and integrate CoW faults
	properly with existing fault handling.

Memory Layout:
	Had to understand xv6's memory layout thoroughly, especially the
	location of stack and special pages like TRAMPOLINE.
